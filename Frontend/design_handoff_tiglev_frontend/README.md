# Handoff: TIGLEV.COM — сайт автосалона

## Overview
Сайт автосалона TIGLEV.COM: главная страница, каталог автомобилей с бесконечной прокруткой, карточка товара (детали авто), форма продажи/выкупа авто, страница «Авто на заказ» (из Азии/Европы), контакты. Общий Header переиспользуется на всех страницах.

## About the Design Files
Файлы в `designs/` — **дизайн-референсы, сделанные в HTML** (Design Components этой среды: `<x-dc>`, шаблонные holes `{{ }}`, `sc-for`/`sc-if`, `dc-import`). Это не production-код для копирования один в один. Задача разработчика — **воссоздать эти экраны в существующем стеке проекта** (Next.js 16 + React 19 + Tailwind CSS 4, см. `src/` в репозитории `wolonee/TiglevWebsite`), используя его архитектуру, роутинг и паттерны компонентов, а не вставлять HTML-файлы напрямую.

Репозиторий `wolonee/TiglevWebsite` уже содержит частичный каркас (`src/app/page.tsx`, `src/components/Header.tsx`, `Hero.tsx`, `Services.tsx`, `Catalog.tsx`, `Contacts.tsx`, `Footer.tsx`, `WhyUs.tsx`, `CarCard.tsx`, `data/cars.ts`) — часть страниц из этого хендоффа (Каталог, Контакты, Услуги) там уже начаты, но не полностью соответствуют текущей, более проработанной версии в этом хендоффе (карточки услуг, блок «Местоположение» с фото автосалона, hero и т.д.). Нужно **свести существующий каркас к актуальному дизайну** из этого пакета, а недостающие страницы (CarDetail, SellCar, ImportCars) — создать с нуля по App Router (`src/app/catalog/[id]/page.tsx` и т.п.).

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы и интерактивные состояния (hover, loading, formsuccess) — воссоздавать пиксель-в-пиксель насколько возможно средствами Tailwind/React в существующем стеке.

## Screens / Views

### 1. Header (`Header.dc.html`) — переиспользуемый на всех страницах
- Фиксированная шапка, при скролле — смена фона/тени (см. `isScrolled` state).
- Логотип TIGLEV (SVG, `public/logo.svg`, красный `#C41E24`).
- Пункты меню: Главная, Каталог, Продать авто, Авто на заказ. Активный пункт подчёркнут (`2px` красная линия снизу).
- Правый блок: телефон `+7 (8482) 750-750` (ссылка `tel:`), кнопка-CTA «Написать нам» → `Contacts.dc.html`.
- Мобильное меню: бургер → выпадающая панель с теми же пунктами + телефон + CTA.
- Проп `active-page` определяет подсветку активного пункта (используется как `<dc-import name="Header" active-page="...">`).

### 2. Главная (`Tiglev.dc.html`)
- **Hero** (`min-height:100vh`, тёмный фон `#0F172A`): фоновое фото автомобиля (`background-size:cover`), затемняющий градиент, заголовок «Автомобили <акцент>», подзаголовок, CTA-кнопки.
- **Услуги** (`#services`, фон `#F8FAFC`): 3 кликабельных карточки-виджета (вся карточка — `<a>`, не только кнопка): «Каталог автомобилей» → Catalog, «Выкуп автомобилей» → SellCar, «Авто на заказ» → ImportCars. Иконка в цветном кружке, заголовок, описание, CTA-текст со стрелкой прижат к низу карточки (`margin-top:auto`, flex-column). Hover: подъём `-6px` + тень.
- **О нас / Преимущества** (`#about`, тёмный фон `#0F172A`, точечный паттерн фона).
- **Контакты** (`#contacts`, фон `#F8FAFC`):
  - Верхний блок (2 колонки): слева — список контактов (адрес, телефон, режим работы, мессенджер, email — с иконками), справа — форма заявки (сабмит → состояние успеха).
  - Заголовок секции «Местоположение» (эйбрау «Где нас найти» + h2, по центру, отступ сверху 50px).
  - Нижний блок (2 колонки): слева — фото автосалона (одно большое 4:3 сверху + два поменьше 4:3 снизу, без подписи, без отступов между фото — выглядят единым полотном, `gap:2px`, скруглены только по внешнему контуру блока), справа — карта (Yandex Maps embed по адресу, растянута на всю высоту блока слева).
  - Фото салона: `autosalon_1.jpg`/`autosalon_2.jpg`/`autosalon_3.jpg` (порядок на экране: главное — autosalon_2, второе — autosalon_1, третье — autosalon_3) — см. `Component.SALON_PHOTOS` в JS.

### 3. Каталог (`Catalog.dc.html`)
- Заголовок + счётчик «N автомобилей».
- Фильтры (по типу кузова и т.д.) в одном блоке с сеткой карточек.
- Карточки автомобилей: фото (`aspect-ratio:16/10`), кнопка «Просмотр» в правом нижнем углу фото → `CarDetail.dc.html?id=...`.
- **Бесконечная прокрутка**: подгрузка следующей порции при приближении к концу страницы (`IntersectionObserver`/scroll listener, порог `600px` до низа), без кнопки «Загрузить ещё».

### 4. Карточка автомобиля (`CarDetail.dc.html`)
- Хлебная крошка / кнопка назад в каталог.
- Заголовок «Марка Модель» + цена (`#C41E24`, крупный шрифт).
- Галерея: одно большое фото + сетка миниатюр (6 фото), активная миниатюра — рамка `2px solid #C41E24`, клик переключает главное фото, есть навигация стрелками и счётчик «X / 6».
- Слева — «Характеристики» (список spec-строк), под галереей — «Описание» (текстовый блок, широкий, `max-width:800px`).
- Состояние «не найдено»: заголовок + текст + кнопка возврата в каталог.

### 5. Продать авто (`SellCar.dc.html`)
- Hero-заголовок «Выкуп автомобилей» / «Продать авто».

### 6. Авто на заказ (`ImportCars.dc.html`)

### 7. Контакты (`Contacts.dc.html`)
Структура идентична блоку «Контакты» на главной странице (пп. Tiglev.dc.html выше) — это **тот же контент, шаблонизированный отдельно на своей странице**: верхний блок (список контактов + форма), заголовок «Местоположение», нижний блок (фото автосалона + карта).

## Interactions & Behavior
- Header: `isScrolled` меняет фон/тень при скролле; мобильное меню открывается/закрывается по клику на бургер.
- Услуги-карточки: весь виджет кликабелен (`<a>` оборачивает всё), hover — подъём + тень.
- Каталог: infinite scroll подгружает карточки без явной кнопки; фильтры пересчитывают видимый список и счётчик.
- CarDetail: клик по миниатюре или стрелкам меняет активное фото галереи (локальный state `activeImageIndex`).
- SellCar: загрузка нескольких файлов → превью-миниатюры; сабмит формы → экран успеха.
- Contacts/Home: сабмит формы заявки → экран успеха (аналогично SellCar).
- Все ссылки между страницами — обычные `<a href="Xxx.dc.html">`; в реальном приложении это роуты Next.js App Router (например `/catalog`, `/catalog/[id]`, `/sell`, `/import`, `/contacts`).

## State Management
- Header: `isScrolled` (boolean), `isMobileMenuOpen` (boolean).
- Catalog: `catalogVisibleCount` (число показанных карточек, растёт при скролле), `catalogLoadingMore` (boolean), выбранные фильтры.
- CarDetail: `activeImageIndex` (индекс активного фото в галерее).
- SellCar / Contacts / ImportCars formы: поля формы (модель, год, тип кузова и т.д. для SellCar; имя/телефон/сообщение для Contacts) + `isFormSuccess`/`isNotSuccess` (boolean после отправки).
- Данные по авто (`Component.CARS`) сейчас статичный массив в JS — в реальном приложении должен приходить из API/БД (см. `src/data/cars.ts` в репозитории — уже есть заготовка, нужно свести к актуальной структуре полей: brand, model, year, price, image, specs[], gallery).
